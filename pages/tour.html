Comments

All developers strive to make their code understandable and readable. Sometimes it may be necessary to make a statement or make a note. Comments serve a variety of purposes, such as commenting, annotating, and taking notes in code. It is a convenient tool to understand the algorithm when you look at it later or when a different developer looks at it.

Singleline Comment

// is used for singline comments. The rest of the line from the point of use is marked as a comment.

Multiline Comment

/* ... */ are used for multiline comments. It accepts the part from opening to closing as a comment.
/* Say hello to name. x = Name. */ fn Hello(name) { println('Hello ' + name) }
It is always advantageous to use a single line comment because the lexer it gives free space to the memory by deleting these comments from the lines that are read.

    Variables

    Fract is immutable by default. This is one of the approaches to the safe and clean programming experience that Fract brings to you. However, this is not a requirement. You can optionally use it as a mutable. Variables are a convenient way to remember data and can take on various types of values. We mentioned this in data types.
    • A value must be given when defining a variable.
    • Variable names are must comply to naming conventions.


    The var keyword used for declare variable.
    var x = <@expression>
    The var keyword is immutable. If wants declare mutable variable, use mut keyword.
    mut x = <@expression>
    You may want to create a variable whose value cannot be changed after it is defined. The const keyword used for declare constant variable, is immutable and cannot set as mutable.
    const pi = 3.14
    Variable declarations can be provided by grouping.
    var ( FiboFirst = 1 Ln2Hi = 6.93147180369123816490e-01 Hex = 0x07EDD5E59A4E28C2 )

    Short Variable Declaration

    It is possible to define variables by shortcut, but this way does not allow marking the defined variable as mutable, const or protected.
    x := <@expression>
    The definition above is equivalent to defining with the var keyword.

    Setting Variables

    Defined variables can be changed later. There is a pretty easy way to do this. You write the new value you want to give after the name of the variable.

    Lists

  • List Object

    • The list structure, which is useful for holding multiple values, is offered by many programming languages. Once an list is defined, its size remains constant. Each element can be of a different data type.
    list := [1, 20.3, 'Fract', print]
    An list is defined by writing the values ​​between the parentheses, separated by commas.

    list := [1, 10, 2, 20, 3, 30] println(list) println(list[2]) list[2] = 200 println(list) // OUTPUT // [1 10 2 20 3 30] // 2 // [1 10 200 20 3 30]
    List allows to get and set values ​​with the help of index. The index is specified as an integer in parentheses and access is provided. Index 0 represents the first element, and it continues to increase by one for each subsequent element. It is also possible to access elements in reverse with negative indexing. Negative indexing starts with -1 and continues to decrease one by one for each subsequent element.

    list := [1, 10, 2, 20, 3, 30] println(list[-2]) // OUTPUT: 3
    The index -2 of the list defined above is equal to index 4 by reverse indexing. Since -1 represents the last index value, each decrease represents the index before it. In other words, -2 index allows you to get the second-to-last element.

    Multiple items can be selected with the help of an list.
    e9 := [89, 189, 289, 389, 489, 589] e9[0] = [90, 19, 20] println(e9) e9[[0, 2, 3, -1]] -= 89 println(e9) println(e9[[-1, -2, -3]]) e9 = 'Fract Programming Language!' println(e9[[0, 1, 2, 3, 4, 5, -9, -8, -7, -6, -5, -4, -3, -2, -1]]) /* OUTPUT [[90 19 20] 189 289 389 489 589] [[1 -70 -69] 189 200 300 489 500] [500 489 300] Fract Language! */
    Elements are selected and processed in multiple ways with the index values ​​given in the list. It's just like a normal indexing, only this time you're selecting multiple elements.

    Why Immutability?

    By default, mutable values ​​can cause undesirable results. Variable values ​​represent a reference when used directly, and the same value now has more than one delegate. This may cause you to change a value that you do not want to change.
    a := [1, 4, 8] b := a b[2] = 10 println(a) println(b) // OUTPUT // [1 4 8] // [1 4 10]
    Since it is immutable by default, there is no referencing case here. Variable b takes the value of variable a directly, it does not reference variable a . Therefore, when the second index is changed on variable b , this change is not reflected in variable a .
    mut a = [1, 4, 8] b := a b[2] = 10 println(a) println(b) // OUTPUT // [1 4 10] // [1 4 10]
    When you create a variable as mutable, it now means that it is referenceable. Now variable b takes the reference of variable a instead of getting its value directly. Therefore, changes on b are directly reflected on variable a . In fact, you are changing variable a by using variable b .
    An immutable variable that references a mutable variable, when given as a value to another variable, copies the value instead of giving a reference to the value it refers to. In order for it to reference, the reference variable must also be created as a mutable.
    If somehow the value needs to be passed as mutable, but the variable you're storing is not mutable, you don't have to worry. Every value you mark as mutable is copied as mutable.
    a := [1, 100, 1000, 10000] b := mut a


    Control Flow

    It is the building block of most programming languages ​​to repeatedly run or manipulate algorithms according to conditions. The most common expressions that will allow you to control the Fract code are if and loop expressions.

    if Expressions

    An if statement allows you to branch your code based on conditions. You provide a condition and then write the algorithm that will run if the condition is met. Optionally, you can also specify alternative conditions and situations where no conditions are met.
    n := 3 if n < 5 { println('condition is true') } // OUTPUT: condition is true
    Each conditional statement begins with an if by the condition. Then, the algorithm that will be realized if the condition is met is written into the block.

    n := 50 if n < 0 { println('n is negative') } else if n < 100 { println('n is less than 100') } // OUTPUT: n is less than 100
    You can add else if statements after the if expression for alternative conditions. It specifies an alternative condition that works if the previous conditions are not met.

    n := 300 if n < 0 { println('n is negative') } else if n <= 100 { println('n is less than 100') } else { println('n is greater than 100') } // OUTPUT: n is greater than 100
    If you want to specify an algorithm that will run if no conditions are met, you can use the else statement. This statement is written at the bottom and should be the last statement of the block. It works when no previous condition is met.
    Preferably, you can also use 0 instead of false and 1 instead of true as the keyword true represents 1 and the keyword false represents 0.

    Loops

    It is useful to execute a block of code more than once. Fract provides various loops to do this. A loop repeatedly executes the code block inside it as the condition is met.

    Fract has two different cycles: foreach and while loops.

    While Loop

    While the condition is true, the loop runs.
    n := 0 for n < 10 { n += 1 println(n) }
    The loop has a condition that will run as long as the variable n is less than 10. During the loop, the variable n is incremented at each iteration, when it reaches 10, the loop ends as the condition is no longer met.
    The best way give an empty condition for infinite loop.

    Foreach Loop

    You can loop through the elements of an list with a while loop.
    t1 := [0, 3, 2, 1, 90] i := 0 for i < 5 { println(t1[i]) i += 1 }
    As expected, this code will work correctly. The loop is set to rotate according to the length of the list and will print each element to the screen. However, this approach is error-prone and requires a lot of code modification. If the size of the known list changes in the future, this code will cause an error if its size is less than 5, and if it is large, it will print incompletely. So in any change the condition also needs to be updated, forgetting to change it will cause problems.
    t1 := [0, 3, 2, 1, 90] for i in t1 { println(t1[i]) }
    You can eliminate these situations with the foreach loop. It adjusts itself according to the size of the list, and each element runs as a single repeat. Also, the code now looks cleaner and more readable.

    The first parameter of the loop always returns an index of current element. If you want to get the element itself, you can add a second parameter. If you don't need to use the index value, you can ignore it.
    t1 := [0, 3, 2, 1, 90] for _, i in t1 { println(i) }
    The code now looks cleaner and performs better. When the element is accessed with index, the element is read again from the memory according to the index value, but when you add the element parameter, you do not need to do an extra reading as it will already be pulled from the memory.
    Can be use := operator for place of in keyword.

    Loop Controlling

    In various situations you may want to manage loops such as ending loops or switching to other loop iterations and this is a very useful method that is widely used.
    counter := 0 for counter <= 10 { counter = counter + 1 if counter > 5 { break } print(counter, ' ') } // Output: 0 1 2 3 4 5
    With the keyword break , it is possible to override and terminate the entire loop even when the loop could still run.

    for _, item in [ 0, 1, 2, 3, 4.0 ] { if item == 1 || item == 3 { continue } print(item, ' ') } // Output: 0, 2, 4
    The keyword continue is used to pass the loop to the next iteration. If there is no next loop iteration, the loop is finished.

    Functions

    Functions are very useful for adding functionality to your code, functions are very common in Fract code. Functions are declared with the fn keyword.
    fn helloWorld() { println('Hello World!') } helloWorld()
    This code is a simple function declaration. It can be called without parameters and write "Hello World!".
    Parentheses are not required if the function takes no parameters.


    Functions can also be defined to have parameters, which are special variables that are part of a function’s signature. When a function has parameters, you can provide it with concrete values for those parameters.
    If a parameter is equal to a different definition name, it will replace that definition.
    fn sum(a, b) { println(a + b) } sum(5, 4) // OUTPUT: 9
    The above function takes two values, sums them up and writes them to the screen. Since the values ​​given as arguments are 5 and 4 , the a parameter is 5 and the b parameter is 4 , respectively.

    A parameter can take a default value. Parameters that take a default value do not have to take a value.
    fn sum(a, b=90) { println(a + b) } sum(5) sum(2, 10) // OUTPUT // 95 // 5.0
    Although no value was specified during the first call, 90 was given instead of the b parameter, which defaults to 90 . Because a custom value was specified in the next call, the value 90 was overridden and the value of the argument became 10 .

    Arguments can also be specified as names and their order can be ignored.
    fn divide(a, b, c) { println((a + b) / c) } divide(c=9, a=5, b=0) divide(2, c=10, b=0) // OUTPUT // 0.5555555556 // 0.2
    After an argument is specified by name, all subsequent arguments must also be specified by name. In the first call, all the arguments were specified in different order by name and received their values. In the second call, the first argument is not specified with a name. All values ​​until specified by name are assigned to the arguments in order, like a regular call.

    A parameter can be used with the ... operator so that it can take multiple values. In this way, the parameter will take all values ​​as an list.
    fn params(...a, ...b=[]) { r := 0 for _, n in a { r += n } for _, n in b { r += n } println(r) } params(100, 110, 120) params(1, 4, 2, 6, b=0, 2, 5, 10) // OUTPUT // 330 // 30
    The code above has a function with two more than one parameter, and this function finds the total values ​​of the parameters, adds them together and prints them out. The first call assigns all values ​​to parameter a , parameter b remains its default value, the empty list. If a parameter that can take multiple values ​​is followed by another parameter, it must be specified with its name. As was done in the second call, the a and b parameters now have special values.

    It can also be used when giving an argument to a parameter with this notation. While giving arguments, it sends the elements of the list one by one, not the list itself.
    println(['Fract', 'Programming', 'Language']) println(['Fract', ' ', 'Programming', ' ', 'Language']...) /* OUTPUT [Fract Programming Language] Fract Programming Language */


    Functions can also return a value.
    fn asHello(name) { ret 'Hello ' + name + '!' } println(asHello('Fract')) // OUTPUT: Hello Fract!
    The above function converts the value given to it to a hello message and returns it as a value. The ret keyword is used to return a value. When the function returns a value, the function returns the value and stops executing, if there is any code that follows it, they will never be reached.
    If function is not return any value, returns none.

    Anonymous Functions

    Anonymous functions can be easily defined. It's not much different from defining a normal function. Anonymous functions can be very useful and a good choice.
    b := fn(x) { println('Hello ', x, '!') } b('Fract') // OUTPUT: Hello Fract!
    The example above is similar to passing a function as a value to a variable. The only difference is that the function is currently defined as a direct value.

    Error Handling

    In some cases, Fract code may panic, preventing current code execution. Catching errors and handling them on a case-by-case basis is an approach offered by many programming languages. Error catching in Fract is done with try-catch blocks.
    try { arr := [1, 2, 3, 10, 341, 0] println(arr[6]) // Out of range. } catch { println('Panic!') }
    You write the codes you want to debug in the try block, in case of a panic, the catch block will run.
    If you don't want use catch blocks, you can use only try block.
    If you want catch panic message, give variable name to catch block.
    try { ... } catch p { ... }

    Deferred Calls

    A defer statement defers the execution of a function until the block ends. The deferred call's arguments are evaluated immediately, but the function call is not executed until the block end.
      Each of these is assumed as an independent block for defer;

    • Main program body
    • Try-Catch body
    • Function body
    defer println('world') println('hello') // OUTPUT // hello // world
    The "hello" is printed first because "world!" the print process waits for the block to finish before being called.

    println('counting') for _, i in range(0, 9) { defer println(i) } println('done') // OUTPUT // counting // done // 9 // 8 // 7 // 6 // 5 // 4 // 3 // 2 // 1 // 0
    Deferred function calls are pushed onto a stack. When a function returns, its deferred calls are executed in last-in-first-out order.

    Concurrency

    Fract offers language-level concurrency. The go keyword is used to perform function calls concurrently. Concurrent calls are performed by creating a lightweight thread managed by Fract.
    Goroutines run in the different memory space.
    fn say(s) { for _, i in range(0, 4) { println(s) } } go say("world") go say("hello") for _, a in range(10, 100) { println(a) }


    Contains Checking

    Often times, it may be necessary to quickly check the value. Checking whether there is a value in it is one of them, Fract has an approach that provides convenience in this regard. With the in keyword, it is also possible to check values ​​inside enumerable values. The value on the left is searched within the value on the right.
    if 'urk' in 'Turkey' { println('Yes!') } else { println('No!') } // OUTPUT: Yes!
    In this code, the value on the left is a string value. The value on the right must also be a string. It is case sensitive and will return true if the value is in it and false otherwise. Since urk value is in Turkey , it returns true.

    if ['Hola', 'Bojour', 'Selam', 'Hello'] in 'Hi' { println('Yes!') } else { println('No!') } // OUTPUT: No!
    An list can be given if it wants to be checked with more than one value. All elements of list must be strings. Since there is no value equal to or containing any value in the list in this code, the result is evaluated as false.

    if 4 in [4, 2, 7] { println('Yes!') } else { println('No!') } // OUTPUT: Yes!
    The left value can be a string, the value given on the right is compared with its elements of list and evaluates to true if it is equal to any of its elements, false otherwise. In the code 4 returns true because it exists in the given list.

    if [4, 'Fract'] in [4, 2, [4, 'Fract'], 7] { println('Yes!') } else { println('No!') } // OUTPUT: Yes!
    When the left value is an list, the value of the other side is searched directly in it, as with string values, when the right value is an list, it does not scroll through all its elements. [4, 'Fract'] evaluates to true because it is contained in the given list. The order of the elements makes a difference in the comparison, if the value was ['Fract', 4] the evaluation would result in false.

    Structs

    Struct is a collection of fields. Used to access fields from the struct instance with a period.
    The keyword struct is used to define a struct. A struct can only contain field names.
    struct Vertex { Lat Long }
    The default constructor function is used to create a struct instance. All fields, including custom fields, must be specified as parameters in the constructor function.
    v := Vertex(10, -10) println(v.Lat, ' ', v.Long) // OUTPUT: 10 -10

    Anonymous Structs

    A classic struct statement can be used anonymously to create anonymous structures without name.
    s := struct{Lat, Long} v := s(10, -10) println(v.Lat, ' ', v.Long) // OUTPUT: 10 -10


    Classes

    Classes can be thought of as a more advanced variant of structures. It contains variables and methods. It can also have a constructor method. The class keyword is used to define a class.

    If a constructor method is not defined, a default constructor method that takes no parameters is defined. To define a constructor method, it is sufficient to define a method in the name of the class.
    class employee { var name = '' var surname = '' var salary = '' fn employee(name, surname, salary) { this.name = name this.surname = surname this.salary = salary } fn getName { ret this.name } fn setName(name) { this.name = name } fn string { ret this.name + ' ' + this.surname + ' ' + string(this.salary) } } e := employee('Daniel', 'McVollo', 9000) println(e.string()) println(e.getName()) e.setName('Dan') println(e.getName()) println(e.string())
    The this keyword is used within the class to access the elements of the class. It is a mutable reference of Class.
    In most programming languages, the keyword this (or equivalent keyword) is not mandatory unless there is a name conflict. However, Fract requires this use for readability, class fields cannot be directly accessed by name within the class.

    Packages

    Every Fract program is made up of packages. Packages allow code to be segmented and organized. Package names must be specified with the package keyword at the beginning of the code. These package names are used for access when packages are imported.
    It is recommended that the package names be the same as the names of the directories they are in.
    Each directory can contain only one package.
